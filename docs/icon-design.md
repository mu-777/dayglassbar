# アイコン設計の経緯（決定記録）

DayGlassBar のアプリアイコン/ロゴをどう決めたかの記録。**なぜこの形にしたか**と、**どの選択肢をなぜ捨てたか**を残し、将来トグルや砂時計へ逆戻りしないためのガードとする。最終確定した仕様は `docs/design.md`「アイコン」節、確定前の検討はここ。（2026-06）

## 結論
- 「**画面の端に灯る静かな光**」というアプリの人格を表す抽象記号。クール青の端の光の筋（sliver）× ダークな"画面"フィールド。
- 筋は**下が濃く・上が薄い** = アプリの実挙動（塗りは下へ縮み、残りが下に溜まる。`README` の「左/右＝下へ」）に向きを合わせる。
- Windows トレイ（カラー）はアプリアイコンの縮小版にして、**実行時のトレイをエクスプローラ/インストール済みアイコンと一致**させる。macOS の template は黒+α一色のため同じ端の筋（右寄り・下が濃い）へ簡略化する。生成は `tools/gen-icons.mjs`（`npm run icons`）。

## 原則（この決定の背骨）
1. **アイコンの仕事は識別であって説明ではない。** 「残り時間が減る仕組み」を図示しようとしない。静止画で"減る"は描けないし、それは説明＝アイコンの誤用。意味は使ううちに後から結びつく。
2. **カテゴリの常套記号を避けることが差別化。** 時計/砂時計/進捗バー/トグルに似せると「ただのタイマー」に埋もれる。本アプリの核は"急かさない"（不変条件#4）で、競合（ポモドーロ等）と真逆。
3. **名前との整合は、絵で名詞を描くことではない。** DayGlass だからといって砂時計を描かない。整合はワードマークとの併置とトーンで作る。"glass/day = 光・透明" の語感が、名前とアプリ（端に灯る光）を橋渡しする。
4. **16px・モノクロで成立する単一シルエット。** 時間ニュアンス（グラデ等）はカラーにしか十分に乗らず、macOS の黒+α template では純粋な筋に劣化する — それは想定どおり（カラーが使える Windows トレイはアプリアイコンの縮小版で意匠を保つ）。

## 紆余曲折（時系列）
各段階の比較画像は下表のシートに保存（生成スクリプトも再生成用に残置）。

1. **出発点＝トグル絵の否定。** 旧アイコン（角丸スレート＋青ピル62%）は iOS トグル/進捗バーに見え、時間の含意が無く、スレート on スレートで小サイズの視認性も低かった。
2. **リテラル案ラウンド**（[assets/candidates/_contact-sheet.png](../assets/candidates/_contact-sheet.png)）: 砂時計 / 砂時計×バー ハイブリッド / 画面の縁バー / 一日の弧。→ いずれも"機能の説明"寄り・常套記号で却下。ここで原則1〜3を確立。
3. **「光のマーク」ラウンド**（[assets/marks/_marks-sheet.png](../assets/marks/_marks-sheet.png)）: M1 端グロー / M2 ひと筋 / M3 光の縁取り / M4 隅の光 / **M5 端のひと筋（M1×M2 合成）**。人格＝周縁の光へ方針転換。
4. **決勝**（[assets/marks/finalists/_finalists-sheet.png](../assets/marks/finalists/_finalists-sheet.png)）: m5（端の筋・推し）/ m3（光の縁取り・小サイズ最強だが汎用的）/ m2（中央の筋）。色温度はクール青に決定（既存バーUIとの連続性）。
5. **「時間の経過要素は要るか」の問い**（[time](../assets/marks/time/_time-sheet.png) / [time-all](../assets/marks/time-all/_time-all-sheet.png)）: v0 なし → v1 経過を匂わせる（非メーター）→ v2/v2b 今の瞬間 → v3 残りを見せる（メーター）。ここで重要な緊張が判明: **「残り時間を素直に描く＝塗りの量＝メーター＝最初に嫌ったトグルの見た目」**。
6. **v3 採用＋向きの反転**（[assets/marks/final/_final-sheet.png](../assets/marks/final/_final-sheet.png)）: 作者は実挙動との一致を重視して v3（メーター）を選択。ただし初期 v3 は「上が濃い」で実挙動と逆だったため、**下が濃い**へ反転して確定。

## 不採用にした選択肢（逆戻りガード）
- **トグル/進捗バー風**: 出発点の失敗。塗りの量を見せると必ずここに戻る点に注意。
- **砂時計（名前の文字どおりの回収）**: "待ち/ローディング"の手垢があり、機能の説明寄り。名前整合はワードマークで担保する方針。
- **時計/カウントダウン/赤/点滅**: "急かす"表現で不変条件#4と衝突。
- **純ワードマークをアプリアイコンに**: 16px で破綻。ワードマークは見出し用の別アセットとして扱う。
